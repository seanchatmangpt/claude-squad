# Documentation Skill

Specialized skill for creating and maintaining comprehensive project documentation.

## Documentation Types

### API Documentation
- Generate Godoc comments for public APIs
- Document function signatures and parameters
- Include usage examples
- Document error conditions

### Architecture Documentation
- Explain system design decisions
- Document data flow between components
- Describe key algorithms
- Include system diagrams

### User Guide
- Installation instructions
- Quick start examples
- Configuration reference
- Troubleshooting guide

### Developer Guide
- Setup development environment
- Running tests and checks
- Commit conventions
- Pull request process

### Changelog
- Document new features
- List bug fixes
- Note breaking changes
- Include migration guidance

## Quality Standards

### Completeness
- All public functions documented
- Examples are runnable
- Links are up-to-date
- Code samples match current implementation

### Clarity
- Clear, concise language
- Active voice preferred
- Jargon explained
- Visual diagrams for complex concepts

### Accuracy
- Documentation matches code
- Examples work correctly
- References point to valid sections
- No outdated information

## Usage

Generates and maintains documentation automatically. Reviews docs for accuracy when code changes. Updates cross-references and examples. Ensures consistency across documentation.

## Output

Updated markdown files with proper formatting, linked examples, and verified accuracy.
